PLAN DE DESARROLLO: EMULADOR DE BATALLAS POKÉMON (DISTRIBUCIÓN PARA 3 PERSONAS)

Este plan organiza el desarrollo del servidor backend en Java utilizando Sockets TCP y PostgreSQL [cite: 4, 5]. La estructura sigue el estándar de Maven/Gradle para separar la lógica de negocio [cite: 12, 13].

---
FASE 0: CIMENTACIÓN (TRABAJO EN EQUIPO)
---
Antes de la división individual, el equipo debe asegurar:
1. Configuración de Base de Datos: Ejecutar el script 'postgre (3).sql' para levantar las tablas de tipos, especies, estadísticas y movimientos [cite: 31, 34, 38, 42].
2. Estructura de Paquetes: Crear los paquetes principales en el proyecto: com.proyecto.pokemon.network, .engine, .models, y .database [cite: 13, 15].
3. Conectividad: Definir el archivo 'application.properties' o '.env' con las credenciales de la DB y el puerto del servidor [cite: 18].

---
ROL 1: ADMINISTRADOR DE DATOS (MODELOS Y BASE DE DATOS)
---
Responsabilidad: Asegurar que la información fluya correctamente entre PostgreSQL y la memoria de Java [cite: 15].

Tareas Principales:
- Creación de POJOs: Implementar las clases User.java, PokemonLive.java y Move.java en el paquete 'models' [cite: 15].
- Gestión de Conexiones: Desarrollar 'DatabaseManager.java' integrando HikariCP para manejar el pool de conexiones [cite: 10].
- Implementación de DAOs: Crear 'UserDAO' y 'PokemonDAO' para encapsular las consultas de login y carga de equipos [cite: 15].
- Consulta de Equipo: Desarrollar la query compleja con JOINs para extraer el equipo de 6 Pokémon de un entrenador [cite: 75, 76, 77].

---
ROL 2: COMUNICADOR DE RED (NETWORK)
---
Responsabilidad: Gestionar las conexiones entrantes y la estandarización de mensajes entre cliente y servidor [cite: 15].

Tareas Principales:
- Servidor de Sockets: Implementar 'Server.java' para escuchar conexiones TCP y 'ClientHandler.java' para manejar cada jugador en un hilo independiente [cite: 4, 7, 15].
- Protocolo de Comunicación: Definir en 'Protocol.java' la estructura de los mensajes JSON para acciones como SEND_MOVE y UPDATE_STATE [cite: 15, 25, 27].
- Matchmaking: Programar la lógica inicial de Handshake y asignación de jugadores a salas de batalla [cite: 22].

---
ROL 3: ÁRBITRO DEL JUEGO (ENGINE)
---
Responsabilidad: Procesar la lógica de combate, cálculos de daño y cumplimiento de reglas [cite: 8, 9, 15].

Tareas Principales:
- Gestión de Batalla: Implementar 'BattleRoom.java' para orquestar el flujo de la partida entre dos hilos de red [cite: 15].
- Gestor de Turnos: Desarrollar 'TurnManager.java' para decidir el orden de ataque basándose en la velocidad de los Pokémon [cite: 15, 26].
- Calculadora de Daño: Programar 'DamageCalculator.java' aplicando la fórmula matemática oficial que incluye tipos, IVs, EVs, STAB y críticos [cite: 15, 26, 69, 70].

---
PLAN DE ACCIÓN PARA EL DÍA 1
---
1. Dev 1: Configurar la conexión base a PostgreSQL y mapear la clase Entrenador.
2. Dev 2: Levantar el socket básico en el puerto elegido y recibir un mensaje de prueba desde un cliente.
3. Dev 3: Crear una función que reciba dos tipos elementales y devuelva el multiplicador de daño correcto [cite: 31, 33].

---
NOTAS TÉCNICAS RELEVANTES
---
- Autoridad Única: Todos los cálculos se realizan en el servidor para evitar trampas [cite: 9].
- Persistencia de Combate: Solo se guarda el resultado final y cambios permanentes (experiencia, PP); el clima y estados temporales son volátiles en memoria [cite: 72, 73].
